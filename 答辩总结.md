## 一、研究背景与问题

**背景**：新药研发周期长（10-15年）、投入高（>10亿美元），计算方法辅助的药物-靶标相互作用（DTI）预测可显著缩小实验搜索空间。

**现有方法不足**：
1. 多数以 SMILES 字符串作为药物输入，丢失分子拓扑结构信息
2. 融合策略简单（特征拼接），无法有效建模跨模态交互
3. 离线实验到在线部署之间存在工程鸿沟

**核心研究问题**：
- RQ1：如何设计有效的分子图结构编码方法，具备全局建模能力？
- RQ2：如何设计跨模态融合机制，实现药物-蛋白有效互补？
- RQ3：如何构建从数据到部署的完整系统闭环？

## 二、技术方案

### 2.1 数据

| 项目 | 内容 |
|---|---|
| 数据来源 | BindingDB（280万+条原始记录） |
| 筛选结果 | 20,000 条高质量样本 |
| 标签 | 解离常数 Kd (nM)，取 log10 变换 |
| 划分策略 | 基于 log10(Kd) 分层抽样，8:1:1 |

### 2.2 模型架构

```
分子 SMILES → RDKit → 分子图 (原子特征38维 + 边类型矩阵)
                              ↓
                    Graph Transformer 编码器（4层, 8头, Graphormer风格）
                         - 可学习度编码
                         - 边类型注意力偏置（7种类型）
                         - CLS 虚拟节点
                              ↓
                        分子特征向量 m (256维)
                              ↘
                                门控融合回归头
                              ↗    ├── 门控重标定 g = σ(W[m;p]+b)
                        蛋白特征向量 p   ├── 差分特征 |m_g - p_g|
                              ↑        ├── 乘性交互 m_g ⊙ p_g
                    Protein Transformer 编码器   └── MLP → ŷ = log10(Kd)
                              ↑
蛋白序列 → Token嵌入 + 位置编码 + CLS
```

### 2.3 损失函数

- **SmoothL1 回归损失**：对异常值鲁棒
- **排序损失**：约束预测排序与真实排序一致
- **联合损失**：L = L_reg + 0.1 × L_rank

## 五、主要贡献

1. **分子 Graph Transformer 编码方法**：Graphormer 风格设计，全局自注意力 + 边类型偏置 + CLS节点，解决了传统 GNN 的局部感受野限制和过平滑问题。

2. **门控融合与交互特征**：四元组融合（门控、差分、乘性交互），自适应跨模态权重分配，配合 SmoothL1 + 排序联合损失。

3. **完整技术闭环**：数据治理（BindingDB 280万→20k）→ 模型训练（多种子+消融）→ 系统部署（Web平台+工程优化）。

4. **工程贡献**：三模块集成平台 + 模型缓存/预热/并发/容错 + PyTorch 跨版本兼容方案。

## 六、答辩常见问题预备

### Q1: 为什么选择 Graph Transformer 而不是传统 GNN？

Graph Transformer 通过全局自注意力机制，能够在单层内直接建模任意两个原子之间的关系，避免了 MPNN 多层堆叠导致的过平滑问题。结合边类型注意力偏置，能够显式编码分子的化学键信息。实验表明，在相同参数量下 Graph Transformer 优于 MPNN。

### Q2: 门控融合相比特征拼接的优势是什么？

特征拼接是固定权重的合并，无法区分不同样本中两个模态的相对重要性。门控融合通过可学习的 sigmoid 门控向量，在每个特征维度上自适应分配权重。加上差分特征和乘性交互特征，提供了更丰富的跨模态关系表达。实验中 Pearson 从 0.6583 提升到 0.7587（+15.25%），说明融合策略的有效性。

### Q3: 数据只有 20k 条，是否太少？

20k 是经过严格质量筛选后的高质量样本集（从 280 万原始记录中筛选），质量优先于数量。在此规模下，模型已达到 Pearson 0.76 的稳定表现。未来扩大数据规模（至87k全量有效记录）有望进一步提升性能，这是明确的改进方向。

### Q4: 为什么不直接使用预训练蛋白语言模型（如 ESM-2）？

ESM-2 等大型预训练模型参数量达到数十亿级别，在当前硬件和数据规模下微调代价较大。本文采用从头训练的轻量级 Protein Transformer（约 100 万参数），在 20k 数据上已取得优于 BiGRU 的性能。引入预训练蛋白模型是重要的改进方向。

### Q5: 排序损失的作用是什么？

在药物筛选场景中，候选化合物的相对排序比绝对数值更重要（研究者更关心"哪个化合物更好"而非"具体 Kd 是多少"）。排序损失约束预测排序与真实排序一致，直接优化了 Pearson 相关系数。联合损失兼顾了数值精度和排名可靠性。

### Q6: NaN 问题是怎么产生的？如何解决的？

NaN 问题有两层原因：（1）Padding 位置在注意力计算中因 -∞ softmax 后产生全零权重，但值向量中已是 NaN 时，0×NaN=NaN 导致传播；（2）PyTorch 2.0 的 MultiheadAttention 快速路径将浮点 attn_mask 错误转为布尔类型，破坏了加性偏置的语义。解决方案：手动实现注意力计算 + masked_fill 强制置零。

### Q7: 系统的可扩展性如何？

系统采用模块化设计，支持：（1）更换分子编码器（如替换为 3D 图网络）；（2）更换蛋白编码器（如接入 ESM-2）；（3）更换融合策略；（4）添加新的对照模块。工程层面支持水平扩展（多实例部署 + 负载均衡）。

### Q8: 该系统能否用于实际药物筛选？

当前系统是概念验证阶段的研究原型，适用于科研辅助和教学演示。将其应用于实际药物筛选还需要：（1）更大规模的训练数据和更严格的验证；（2）不确定性估计和置信区间；（3）领域专家的验证和标定。系统输出不构成任何临床决策依据。
